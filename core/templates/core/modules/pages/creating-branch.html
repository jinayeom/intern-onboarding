Creating Your Own Dev Branch

You got your first tivket, and now you have to create your own branch.

Follow these steps to create your own dev branch for the repo.

  1. Ensure git is installed in your VM. Check by running git in your terminal. This should show a bunch of commands; this indicates you have git. If not, you need to download git.
  2. Check to see which branch you are in now. You are probably in the master branch. Run: git branch
  3. Create a branch for your ticket by following the naming convention. Eg if your ticket name is CESVAN-1234 and your task is to create Unite Tests for the front-end, name it as follows: git branch "CESVAN-1234_UTFrontEnd"
  4. Switch to your newly created branch: git checkout CESVAN-1234_UTFrontEnd
  5. Double check you are in the correct branch: git branch You should see the * next to the branch you want to be in.
  6. Before you change anything, ensure you have the most recent version of the repo: git status If you are not up to date, then run the following commands: git checkout master to switch back to the master branch, then git pull.

TIP: To compile your code, check out the "How to Compile Your Code" module.
