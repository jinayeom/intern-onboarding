SSHing into Your Development Environment

Working on a remote server (your Vm eg jina-dev) is basically logging into a remote computer that has a version of the git repo for WebApps installed on its “hard drive”. When the full-timer set up your VM initially, they set it up so when you are working on this CE_WebApps folder, git pull / git push are pulling/pushing to the CE_WebApps repo. You do not have to connect the repo yourself. This would’ve been done for you when your VM was created. AKA it’s only when you git push your files to the repo where your code will show up on the shared repo.

Follow these steps to securely access your remote development environment via SSH.

  1. Open your IDE (eg VS Code) or terminal application. In this example, I will assume you are using VS Code.
  2. Once inside, connect to your remote VM by selecting the two arrows at the bottom left. Enter your IP address at the top search bar. Press "Enter" to join your dev VM.
  3. Once inside, open up your IDE's CLI/Terminal and write the following: laser. Follow it up with code . to open up the folders in the Explorer (left-side panel). TIP: If you are deep in another folder and you want to re-navigate back to this home directory, then you can do so by writing this command: laser
  4. You should have received a development environment link, kind of like your own version of Laser Sandbox so you can see how your dev changes manifest in the product. Navigate to this website.
  5. You will be prompted to login. To do so, you will need to create a superuser. Go to your command linbe and run: python3 manage.py createsuperuser. Create your own username/password, but I suggest you use these credentials that everyone uses: username=bestotesto and password=laserlaser.
  6. Once successfully created, go back to the browser and log in with your account details.

Tip: If you encounter “Permission denied (publickey)” errors, check file permissions on your ~/.ssh folder.
